# Qu'est-ce que Git ?

Git est un système de contrôle de version distribué, utilisé pour suivre les modifications apportées aux fichiers et coordonner le travail entre plusieurs personnes. 
Il permet de gérer les versions de code source, de collaborer avec d'autres développeurs et de maintenir un historique des modifications.

## Principales fonctionnalités de Git

- **Suivi des modifications** : Git enregistre les modifications apportées aux fichiers dans un dépôt.
- **Branches et fusions** : Git permet de créer des branches pour travailler sur des fonctionnalités ou des corrections de bugs de manière isolée, puis de fusionner ces branches dans la branche principale.
- **Collaboration** : Git facilite la collaboration entre les développeurs en permettant de partager des dépôts et de gérer les contributions de chacun.
- **Historique** : Git conserve un historique complet des modifications, permettant de revenir à des versions antérieures si nécessaire.

## Commandes de base

- `git init` : Initialise un nouveau dépôt Git.
- `git clone <url>` : Clone un dépôt distant.
- `git add <fichier>` : Ajoute des fichiers à l'index pour le prochain commit.
- `git commit -m "message"` : Enregistre les modifications dans le dépôt avec un message descriptif.
- `git push` : Envoie les commits locaux vers un dépôt distant.
- `git pull` : Récupère les modifications depuis un dépôt distant et les fusionne avec la branche locale.

Pour plus d'informations, consultez la [documentation officielle de Git](https://git-scm.com/doc).
